# run-logs/ — Raw measurement run log

**Path:** `medicion/run-logs/`
**Belongs to:** [`medicion/`](../README.en.md)

---

## What it is and why it exists

This folder contains the **raw evidence CSVs** for every individual run against n8n's
webhooks (as-is and to-be, Bot and IoT). They are the primary source for the thesis
proposal's Operation dimension latency and failure-rate metrics.

## Contents of this folder

| Subfolder | Description |
|---|---|
| [`bot/`](bot/README.en.md) | Bot case run-logs (as-is + to-be) |
| [`iot/`](iot/README.en.md) | IoT case run-logs (as-is + to-be) |

## Relationship to the methodology

Exact column format: `run_id,case,version,input_set,start_ts,end_ts,status,error_type,notes,commit_hash`
(see [`docs/context/convenios-y-reglas.md`](../../docs/context/convenios-y-reglas.en.md)
§Run Logs). **Critical integrity rule:** once a run is recorded, it is **never deleted or
modified** — if there's a recording error, a new correct row is added and the incorrect
one is marked with `status: invalid`. These CSVs are generated by
`automatizacion/run_corridas.py` following
[`docs/protocolo-evidencias.md`](../../docs/protocolo-evidencias.en.md) §4-5.
