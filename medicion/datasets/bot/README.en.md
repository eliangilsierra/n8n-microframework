# datasets/bot/ — Bot case Input Sets

**Path:** `medicion/datasets/bot/`
**Belongs to:** [`medicion/datasets/`](../README.en.md)

---

## What it is and why it exists

Contains the Bot case's 10 `input-set-{A..K}.json` files, deterministically generated by
`generar_datasets.py` with the seed in `seeds.yaml`.

## Contents of this folder

| Set | Type | Purpose |
|---|---|---|
| `input-set-A.json` | Static | Normal case — valid message, valid token |
| `input-set-B.json` | Static | High load / urgent message |
| `input-set-C.json` | Static | Invalid data — missing token |
| `input-set-D.json` | Static | Boundary values — empty message |
| `input-set-E.json` | Static | Partial fields — missing `user_id`/`session_id` |
| `input-set-F.json` | Dynamic | Normal realism (200 varied messages) |
| `input-set-G.json` | Dynamic | Valid/invalid mix (150/50) |
| `input-set-I.json` | Dynamic | Degradation — decreasing delay 300→50ms |
| `input-set-J.json` | Dynamic | Extreme percentiles |
| `input-set-K.json` | Dynamic | Duplicates — measures idempotency (REG-005) |

## Relationship to the methodology

These sets feed the 8000 measurement runs (10 sets × 2 cases × 2 versions × 200
repetitions/N). Detail on each set and which rule it stresses is in
[`casos-de-estudio/bot/adr/ADR-004`](../../../casos-de-estudio/bot/adr/ADR-004-diseno-experimental-input-sets.en.md).
**Immutable during the measurement phase** — do not edit without an ADR.
