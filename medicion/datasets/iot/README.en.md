# datasets/iot/ — IoT case Input Sets

**Path:** `medicion/datasets/iot/`
**Belongs to:** [`medicion/datasets/`](../README.en.md)

---

## What it is and why it exists

Contains the IoT case's 10 `input-set-{A..K}.json` files, deterministically generated by
`generar_datasets.py` with the seed in `seeds.yaml`.

## Contents of this folder

| Set | Type | Purpose |
|---|---|---|
| `input-set-A.json` | Static | Normal reading, no alert |
| `input-set-B.json` | Static | High temperature, active alert |
| `input-set-C.json` | Static | Invalid data — non-numeric temperature |
| `input-set-D.json` | Static | Boundary values — values exactly at the threshold |
| `input-set-E.json` | Static | co2 missing from the payload |
| `input-set-F.json` | Dynamic | Normal realism (200 varied readings) |
| `input-set-G.json` | Dynamic | 70/15/10/5 industrial mix |
| `input-set-I.json` | Dynamic | Gradual degradation — normal → critical |
| `input-set-J.json` | Dynamic | Extreme percentiles p1/p99 |
| `input-set-K.json` | Dynamic | Duplicates — measures `{sensor_id, timestamp}` idempotency (REG-005) |

## Relationship to the methodology

These sets feed the 8000 measurement runs. Detail on each set and which rule it stresses
is in
[`casos-de-estudio/bot/adr/ADR-004`](../../../casos-de-estudio/bot/adr/ADR-004-diseno-experimental-input-sets.en.md)
(experimental design shared with Bot). **Immutable during the measurement phase** — do
not edit without an ADR.
