# consolidado/ — Final as-is vs to-be comparison reports

**Path:** `medicion/consolidado/`
**Belongs to:** [`medicion/`](../README.en.md)

## What it is and why it exists

This folder contains the **consolidated reports** derived from the raw run-logs and
cr-logs: as-is vs to-be comparisons, derived metrics per dimension (Delivery, Operation,
Security, Traceability), the ATAM evidence matrix, and the MTTD measurement result.

## Contents of this folder

| File | Description |
|---|---|
| [`comparacion-2026-05-03.md`](comparacion-2026-05-03.en.md) | First as-is vs to-be comparison (intermediate snapshot) |
| [`comparacion-2026-05-05.md`](comparacion-2026-05-05.en.md) | Second as-is vs to-be comparison (Phase 6 final snapshot) |
| [`metricas-derivadas.md`](metricas-derivadas.en.md) | Metrics across the thesis proposal's 4 dimensions, with verified indicative goals |
| [`atam-evidencia.md`](atam-evidencia.en.md) | Evidence matrix for the 12 top-K ATAM scenarios (Bot + IoT) |
| [`mttd-resultado.md`](mttd-resultado.en.md) | Mean Time To Detect measurement result (BOT-Q5, IOT-Q4) |
| `reporte-runlogs.html` | Interactive HTML report generated by `medicion/analizar_runlogs.py` (do not version manually — it's regenerated) |

## Relationship to the methodology

**Note:** unlike the raw run-logs/cr-logs (immutable once recorded), the reports in this
folder are regenerated when new evidence becomes available (hence the date suffix on the
comparison files). They are direct input to the final ATAM report
([`docs/atam/informe-atam-final.md`](../../docs/atam/informe-atam-final.md)) and to the
best practices guide
([`docs/guia-buenas-practicas.md`](../../docs/guia-buenas-practicas.en.md), Appendix C).
